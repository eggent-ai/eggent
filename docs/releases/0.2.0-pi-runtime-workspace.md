# Eggent 0.2.0 - Pi Runtime Workspace

Date: 2026-07-22  
Type: Minor release snapshot

## Release Name
`Pi Runtime Workspace`

This release turns Eggent into a Pi-powered local-first workspace with project agents, runtime extension support, schedules, skills, files, Telegram, voice transcription, and external API improvements.

## What Is Included

### 1) Pi Runtime Integration
- Project chat now runs through the Pi SDK runtime.
- Model/provider configuration uses Pi runtime settings and credentials.
- Runtime extensions can be bound for tools, MCP, skills, and web access.

### 2) Projects, Files, and Skills
- Project pages include context, memory, model, MCP, skills, and files workflows.
- File upload, preview, creation, deletion, and drag-and-drop folder targeting were improved.
- Sidebar folder selection no longer changes the agent working directory.

### 3) Schedules and Pipelines
- Legacy cron UI/runtime was replaced by Pi-backed schedules.
- Pipelines can run multi-step agent workflows with artifact handoff.
- Pipeline APIs and run artifact APIs are documented in the README.

### 4) Telegram and External API
- Telegram setup supports polling activation from token setup.
- Telegram replies include progress updates while long agent runs are active.
- External API project lookup and chat/session continuity were improved.

### 5) Voice and Attachments
- Local whisper.cpp speech-to-text supports browser dictation and Telegram voice transcription.
- Runtime/cache/model/audio paths are kept under data directories for easier deployment hygiene.

### 6) Release Hardening
- Docker defaults to localhost binding unless explicitly exposed.
- Docker runtime image excludes source, git metadata, env files, and sourcemaps.
- whisper.cpp is pinned for reproducible Docker builds and ARM64 compatibility.
- Type checking is clean with `npx tsc --noEmit`.

## New in 0.2.0

- Pi SDK runtime integration.
- Pi schedules in place of legacy cron.
- Project skills and improved project workspace management.
- Optional runtime `bundled-skills/` catalogs for deployments that provide prebuilt skills.
- Pipelines and pipeline artifacts.
- Local speech transcription.
- Hardened Docker install defaults and release image guardrails.
- Public repository/install references updated to `eggent-ai/eggent`.

## Upgrade Notes

- No data migration is required for existing `data/` directories.
- Schedules are now stored by the Pi runtime under `.pi/subagent-schedules/` in the relevant context.
- Docker still binds to `127.0.0.1` by default. Set `APP_BIND_HOST=0.0.0.0` only when direct network exposure is intentional.
- Provider API keys can be configured from the UI; `.env` remains optional.
